Difference between var, let and const keywords in JavaScript

The var, let, and const is the keyword to declare variables in javascript.

In JavaScript, var is function-scoped and hoisted, let is block-scoped and not hoisted, while const is block-scoped and ensures immutability for its assigned value. Use const by default for variables that won't change, let when reassignment is necessary, and avoid var due to its scoping and hoisting issues.

In modern JavaScript, it's recommended to use const by default for variable declarations, and only use let when you expect the variable to change its value. var should be avoided due to its scoping and hoisting quirks. Understanding the scope of these keywords is crucial for writing clean and predictable JavaScript code.

  • If a function is called without passing an argument for a parameter, that parameter is undefined


Usecase of undefined :

  • It can be assigned to variables or properties to reset them or remove an existing value.
  • In data structures like arrays or databases, null can be used to indicate an empty or non-existent element or record.

What is rest API

A REST API (Representational State Transfer Application Programming Interface) is a set of rules and conventions for building and interacting with web services over the HTTP protocol. It is an architectural style for designing networked applications, and it relies on a stateless, client-server communication model.
